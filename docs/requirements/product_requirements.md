## 1. 项目概述

LifeQuest 是一个基于 LLM 的游戏化自我成长平台。

项目将用户每天的学习、生活、情绪、作息和任务完成情况转化为角色成长数据，并通过每日结算、属性变化、Buff / Debuff、成长路线、敌人系统、成就称号、剧情旁白等方式，为用户提供类似 RPG / Galgame 的成长反馈体验。

项目目标不是把生活变成游戏，而是帮助用户更直观地看到自己的成长过程，并通过及时反馈获得持续行动的动力。

## 2. 一句话定位

LifeQuest 是一个把现实生活变成「可记录、可反馈、可攻略」的 AI 自我成长平台。

## 3. 目标用户

### 3.1 学生群体

包括考研学生、大学生、高中生等。

主要需求：

- 制定学习计划
- 记录学习状态
- 获得每日反馈
- 提升学习动力
- 发现拖延、熬夜、焦虑等问题

### 3.2 求职准备群体

包括准备实习、校招、转岗的用户。

主要需求：

- 记录面试准备进度
- 管理技术学习路线
- 跟踪项目、简历、投递、模拟面试等任务
- 获得阶段性复盘和改进建议

### 3.3 自我提升用户

包括希望改善作息、运动、情绪管理和生活节奏的用户。

主要需求：

- 养成良好习惯
- 获得生活状态反馈
- 通过游戏化机制提升坚持动力
- 建立长期成长记录

## 4. 核心问题

项目主要解决以下问题：

- 普通打卡工具反馈单一，难以持续激励用户。
- 用户记录了生活数据，但缺少解释和建议。
- 用户无法直观看到自己的长期成长轨迹。
- 学习、生活、情绪、作息之间缺少统一建模。
- LLM 直接评分容易不稳定，需要规则评分与 LLM 解释相结合。

## 5. 核心创新点

### 5.1 现实行为映射为角色成长系统

平台将现实行为转化为角色属性变化：

- 学习行为 -> 知识积累、专注力提升
- 完成任务 -> 执行力、自律提升
- 运动、睡眠 -> 精力、情绪稳定提升
- 熬夜、过度娱乐 -> 精力、专注力、生活平衡下降

用户不是简单完成打卡，而是在培养自己的现实角色。

### 5.2 规则评分与 LLM 解释分离

每日评分不完全交给 LLM，而是由后端规则计算。

LLM 主要负责：

- 解析自然语言日志
- 生成每日反馈
- 生成明日任务
- 生成剧情化旁白
- 生成周报和成长建议

这样可以避免 LLM 评分漂移，同时保留 LLM 的文本理解和个性化表达能力。

### 5.3 游戏化反馈机制

平台引入轻量现实 RPG 机制：

- 每日结算
- 角色属性
- 经验值和等级
- Buff / Debuff
- 成长路线
- 敌人系统
- 成就称号
- 随机事件
- 剧情旁白

目标是增强用户的即时反馈和长期参与感。

## 6. 功能需求

### 6.1 用户模块

#### 6.1.1 用户注册登录

用户可以通过账号密码注册和登录系统。

基础字段：

- 用户名
- 邮箱 / 手机号
- 密码
- 创建时间
- 最近登录时间

#### 6.1.2 用户个人目标设置

用户首次进入系统时，需要选择或填写自己的成长目标。

目标类型：

- 学习考试路线
- 求职面试路线
- 健康生活路线
- 综合成长路线
- 自定义路线

用户可设置：

- 当前目标
- 目标周期
- 每周计划投入时间
- 当前阶段描述
- 偏好的反馈风格

反馈风格：

- 冷静教练风
- 温柔陪伴风
- 毒舌监督风
- 游戏系统旁白风
- Galgame 角色风

### 6.2 每日记录模块

#### 6.2.1 表单记录

记录字段：

- 学习时长
- 工作 / 项目时长
- 睡眠时长
- 运动时长
- 娱乐时长
- 今日情绪
- 今日任务完成率
- 今日完成内容
- 今日问题
- 今日复盘

#### 6.2.2 自然语言记录

用户可以直接输入自然语言日志。

示例：

```text
今天上午复习了 Redis，下午写了项目接口，晚上有点焦虑，刷视频刷了一个半小时，运动了 20 分钟。
```

系统调用 LLM 将其解析为结构化数据：

```json
{
  "study_topics": ["Redis"],
  "project_work": ["项目接口"],
  "mood": "焦虑",
  "entertainment_minutes": 90,
  "exercise_minutes": 20,
  "risk_events": ["晚间分心"]
}
```

### 6.3 评分模块

#### 6.3.1 每日总评分

评分维度：

- 学习成长得分
- 任务执行得分
- 精力恢复得分
- 情绪状态得分
- 娱乐控制得分
- 复盘质量得分

默认公式：

```text
每日总分 =
学习成长得分 * 25%
+ 任务执行得分 * 25%
+ 精力恢复得分 * 15%
+ 情绪状态得分 * 15%
+ 娱乐控制得分 * 10%
+ 复盘质量得分 * 10%
```

#### 6.3.2 每日评级

- S：90-100
- A：80-89
- B：70-79
- C：60-69
- D：40-59
- E：0-39

### 6.4 角色属性模块

系统为每个用户维护一组成长属性：

- 专注力
- 自律
- 知识积累
- 精力
- 情绪稳定
- 执行力
- 生活平衡

每个属性范围为 0-100。

属性变化示例：

- 学习 4 小时 -> 知识积累 +6，专注力 +4
- 运动 30 分钟 -> 精力 +5，情绪稳定 +2
- 熬夜 -> 精力 -6，生活平衡 -4
- 娱乐失控 -> 专注力 -4，自律 -3

### 6.5 游戏化反馈模块

#### 6.5.1 每日结算

每日记录完成后，系统生成类似游戏结算页的反馈。

内容包括：

- 今日评级
- 今日获得经验
- 属性变化
- 触发事件
- 遭遇敌人
- Buff / Debuff
- AI 旁白
- 明日任务

#### 6.5.2 敌人系统

系统将用户常见问题外化为敌人。

敌人类型：

- 拖延史莱姆
- 短视频魅魔
- 熬夜幽灵
- 焦虑雾兽
- 完美主义魔王
- DDL 巨龙
- 三分钟热度哥布林

触发示例：

用户娱乐时间过长且任务未完成时，触发「短视频魅魔 Lv.2」。

#### 6.5.3 Buff / Debuff 系统

正向 Buff：

- 深度专注：连续学习 90 分钟后触发
- 复盘洞察：完成高质量复盘后触发
- 运动恢复：运动 30 分钟后触发
- 晨间启动：上午完成核心任务后触发

负向 Debuff：

- 睡眠不足：睡眠低于 6 小时触发
- 晚间分心：晚上娱乐失控触发
- 空转状态：计划很多但无有效产出触发
- 过载风险：连续多天高强度学习但恢复不足触发

#### 6.5.4 成长路线系统

默认路线：

- 后端实习路线
- 考研复习路线
- 健康生活路线
- 英语提升路线
- 作息修复路线
- 综合成长路线

后端实习路线示例：

1. Java 基础村
2. 数据库矿洞
3. Redis 迷宫
4. 消息队列工厂
5. 高并发战场
6. 项目表达试炼
7. 最终 Boss：技术面试

#### 6.5.5 成就称号系统

称号示例：

- 晨间启动者：连续 3 天上午完成核心任务
- 复盘学徒：累计完成 10 次有效复盘
- 睡眠修复者：连续 5 天睡眠大于 7 小时
- 稳定推进者：连续 7 天完成至少一个主线任务
- 缓存守卫者：完成 Redis 专题学习

称号可以给予轻微属性加成或展示效果。

### 6.6 LLM 反馈模块

#### 6.6.1 每日反馈生成

LLM 根据每日评分、属性变化和用户日志生成反馈。

要求：

- 不重新计算分数
- 不做空泛鼓励
- 必须结合用户当天数据
- 必须给出明确问题和建议
- 语气符合用户选择的反馈风格

#### 6.6.2 明日任务生成

LLM 根据用户目标、今日问题和长期趋势生成明日任务。

任务类型：

- 主线任务
- 支线任务
- 防御任务

示例：

- 主线任务：整理 Redis 缓存三兄弟对比表
- 支线任务：运动 20 分钟
- 防御任务：22:30 后不刷短视频

#### 6.6.3 周报生成

周报内容：

- 本周总评分趋势
- 属性变化趋势
- 最常触发的敌人
- 最有效的正向行为
- 下周建议
- 下周主线任务

## 7. 非功能需求

### 7.1 可用性

系统应提供清晰易用的界面，用户应能在 3 分钟内完成一次每日记录。

### 7.2 响应性能

- 普通页面请求响应时间应控制在 300ms 内。
- LLM 反馈生成可以异步处理。
- 前端先展示规则评分和属性变化，LLM 结果生成后再更新页面。

### 7.3 可解释性

每日评分和属性变化必须有明确依据，不能只展示结果。

示例：

- 专注力 +4：今日学习时长达到 3 小时，且娱乐时间未超过 2 小时。
- 精力 -3：睡眠时间低于 6 小时。

### 7.4 稳定性

当 LLM 调用失败时，系统仍应能展示：

- 每日评分
- 属性变化
- 基础建议
- 明日任务模板

### 7.5 隐私保护

用户日志可能包含个人情绪、生活状态等隐私信息。系统应避免公开展示用户原始日志，并对用户数据进行权限隔离。

### 8.1 前端

- Vue 3
- TypeScript
- Element Plus / Naive UI
- ECharts

### 8.2 后端

- Spring Boot
- MyBatis Plus / Spring Data JPA

### 8.3 数据库

- MySQL：存储用户、任务、评分、属性、路线等结构化数据

- `parseDailyLog`
- `generateDailyFeedback`
- `generateTomorrowTasks`
- `generateWeeklyReport`
- `generateStoryNarration`

## 9. MVP 范围

MVP 必须包含：

1. 用户注册登录
2. 目标路线选择
3. 每日记录
4. 自然语言日志解析
5. 每日评分
6. 角色属性变化

8. LLM 每日反馈
9. 明日任务生成

MVP 暂不包含：

- 好友系统

## 10. 核心用户流程

### 10.1 首次使用流程

```text
注册登录
-> 选择成长目标
-> 选择反馈风格
-> 初始化角色属性
-> 进入成长面板
```

### 10.2 每日使用流程

```text
填写每日记录
-> LLM 解析自然语言日志
-> 系统计算每日评分
-> 系统计算属性变化
-> 生成每日结算
-> LLM 生成反馈和明日任务
-> 用户查看成长结果
```

### 10.3 周期复盘流程

```text
系统汇总一周数据
-> 分析评分趋势和属性变化
-> 统计高频问题和高效行为
-> LLM 生成周报
-> 用户调整下周目标
```

## 13. 项目边界

LifeQuest 不是心理咨询系统，也不是医疗建议系统。

当用户输入明显涉及严重心理危机、健康风险等内容时，系统不应给出诊断或治疗建议，而应提示用户寻求专业帮助。

项目核心能力：

- 生活记录
- 成长分析
- 游戏化反馈
- 学习和生活建议
- 长期趋势复盘

## 14. MVP 验收标准

MVP 阶段验收标准：

1. 用户可以完成注册、登录和目标设置。
2. 用户可以提交每日记录。
3. 系统可以根据记录生成结构化数据。
4. 系统可以计算每日评分。
5. 系统可以更新角色属性。
6. 系统可以生成每日结算页面。
7. 系统可以调用 LLM 生成个性化反馈。
8. 系统可以生成明日任务。
9. 系统可以展示成长趋势图。
10. 系统在 LLM 调用失败时仍能正常展示基础结果。

## 15. 后续扩展方向

- 社交系统：好友、组队、成长小队。
- 可穿戴设备接入：运动、睡眠、心率等真实数据。
- 多 Agent 成长教练：学习教练、情绪陪伴、作息管理、求职规划、游戏旁白等。
- 更复杂的路线分支：稳定成长路线、高压冲刺路线、间歇爆发路线、作息修复路线。
- 插件化成长路线：Java 后端路线、考研数学路线、英语六级路线、减脂路线、论文写作路线等。
